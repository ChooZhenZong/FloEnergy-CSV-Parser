# FloEnergy CSV Parser
**For FloEnergy Tech Assessment**

This program parses FloEnergy CSV files and generates SQL insert statements for meter readings.

---

## Java Version

The program was coded using **Java 25**, but the **minimum Java version required** to build or run it is **Java 9**.

### Java Features Used

| Feature | Minimum Java Version | Notes |
|---------|-------------------|-------|
| Try-with-resources (multiple resources) | 9 | Single-resource try supported since Java 7 |
| `java.time.LocalDate` / `LocalDateTime` / `DateTimeFormatter` | 8 | Modern date-time API for timestamps |
| `BufferedReader` / `BufferedWriter` | 1.1+ | Standard I/O classes |
| Lambda / `String::trim` | 8 | For concise string processing |
| `String.format()` | 5+ | Used for formatting SQL statements |

---

## Build Instructions

From the project root directory, compile the program:

```bash
javac -d bin -sourcepath src src/Main.java
```

- `-d bin` → output compiled classes to the `bin` folder  
- `-sourcepath src` → source code location  

---

## Run Instructions

After compilation, run the program:

```bash
java -cp bin Main "path/to/input.csv" "path/to/output.sql"
```

- `"path/to/input.csv"` → CSV file to parse  
- `"path/to/output.sql"` → SQL output file  

**Defaults** (if arguments are not provided):

- Input: `src/Resource/test.csv`  
- Output: `output.sql` in the project root  

---

## Notes

- The program only generates SQL insert statements; UUIDs are not generated by default.  
- It assumes the CSV contains `200` and `300` record types in order.  
- Large CSV files may need streaming or batch processing to avoid memory issues.  
- Constants for CSV column indexes and table names are defined in `CsvConstants` and `DBConstants` classes for easier maintenance.
